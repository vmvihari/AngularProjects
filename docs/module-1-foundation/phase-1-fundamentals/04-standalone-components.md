# Phase 1, Lesson 4: Standalone Components

If you have worked with older versions of Angular, you are likely intimately familiar with `NgModules` and the massive `app.module.ts` file that controlled your application.

Welcome to modern Angular! **NgModules are dead.** 

As of Angular 14, the framework introduced **Standalone Components**, and as of Angular 17, they are the absolute default for all new applications.

## What is a Standalone Component?

A Standalone Component is completely self-sufficient. It does not need to be declared in an `NgModule`. Instead, the component itself specifies exactly what its dependencies are.

Notice the `standalone: true` flag and the new `imports` array directly inside the `@Component` decorator:

```typescript
import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';

@Component({
  selector: 'app-user-profile',
  standalone: true,
  imports: [CommonModule], // Import what you need, right where you need it!
  template: `<h1>User Profile</h1>`
})
export class UserProfileComponent {}
```

This mental model is much closer to standard JavaScript/TypeScript imports. If a component needs a dependency (like a structural directive or another component), it imports it directly!

## How does the App Boot up?

If there is no `app.module.ts`, how does the application start? Modern Angular uses a completely new bootstrapping API called `bootstrapApplication`.

Instead of an `AppModule`, global application configuration is now provided via an `ApplicationConfig` object, typically found in a file named `app.config.ts`.

## 🎯 Bootcamp Task: Explore Standalone Bootstrapping

Let's look at the actual code generated by Nx to see Standalone architecture in the wild!

### Step 1: Open the App Config
Open `apps/issue-tracker/src/app/app.config.ts`. 

You will see an `ApplicationConfig` object being exported. Notice how global providers are registered using modern helper functions like `provideZoneChangeDetection` and `provideRouter`. This is where we will eventually configure our HTTP client and Global State.

### Step 2: Open Main.ts
Now, open `apps/issue-tracker/src/main.ts`.

Notice how simple it is! The application kicks off by calling `bootstrapApplication()`, passing in the root `AppComponent` and the `appConfig` you just looked at. 

### Step 3: Open the Feature Library Component
Finally, let's look at the library we generated in the last lesson.
Open `libs/issues/feature-manage/src/lib/feature-manage/feature-manage.ts`.

Look at the `@Component` decorator. You will see `standalone: true` and an empty `imports: []` array. Because Nx knows we are building a modern Enterprise app, it generated our feature library using Standalone components automatically!
