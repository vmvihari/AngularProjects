# Phase 8, Lesson 2: The Login Workflow

Now that we have a robust `AuthStore` capable of handling multiple roles, we need to build a UI so users can actually log in!

In an enterprise architecture, we want our login feature to be completely isolated in its own library. This allows it to be easily lazy-loaded without pulling in unnecessary code on initial bundle load.

## Your Task: Build the Login Feature

### 1. Generate the Library
Open your terminal and use Nx to generate a new feature library:

```bash
npx nx g @nx/angular:library --name=feature-auth --directory=libs/shared/feature-auth --standalone
```
*(Note: As always, delete the default `src/lib/feature-auth/` folder generated by Nx!)*

### 2. Create the Login Component
Create a new folder and component at `libs/shared/feature-auth/src/lib/login/login.component.ts`:

```typescript
import { Component, inject } from '@angular/core';
import { Router } from '@angular/router';
import { FormsModule } from '@angular/forms';
import { AuthStore, Role } from '@enterprise-workspace/shared/util-auth';

@Component({
  selector: 'lib-login',
  standalone: true,
  imports: [FormsModule],
  template: `
    <div class="login-container">
      <div class="login-card">
        <h2>Enterprise Login</h2>
        <p>Select a role to simulate a backend authentication flow.</p>
        
        <div class="form-group">
          <label>Select Role:</label>
          <select [(ngModel)]="selectedRole">
            <option value="Admin">Admin</option>
            <option value="Manager">Manager</option>
            <option value="Developer">Developer</option>
            <option value="Viewer">Viewer</option>
          </select>
        </div>

        <button (click)="onLogin()">Login</button>
      </div>
    </div>
  `,
  styles: [`
    .login-container { display: flex; justify-content: center; align-items: center; height: 100vh; }
    .login-card { background: var(--card-bg, #fff); padding: 2rem; border-radius: 8px; box-shadow: 0 4px 6px rgba(0,0,0,0.1); width: 100%; max-width: 400px; }
    h2 { margin-top: 0; color: var(--primary-color, #333); }
    .form-group { margin-bottom: 1.5rem; }
    select { width: 100%; padding: 0.5rem; margin-top: 0.5rem; }
    button { width: 100%; padding: 0.75rem; background: var(--primary-color, #007bff); color: white; border: none; border-radius: 4px; cursor: pointer; }
    button:hover { opacity: 0.9; }
  `]
})
export class LoginComponent {
  public authStore = inject(AuthStore);
  private router = inject(Router);

  // Default selection
  selectedRole: Role = 'Admin';

  onLogin() {
    // 1. Simulate a backend login by passing the selected role to our AuthStore
    this.authStore.login(`Demo ${this.selectedRole}`, this.selectedRole);
    
    // 2. Once logged in, instantly route the user to the Dashboard!
    this.router.navigate(['/dashboard']);
  }
}
```

### 3. Export the Component
Open `libs/shared/feature-auth/src/index.ts` and export the component so the router can find it:

```typescript
export * from './lib/login/login.component';
```

### 4. Wire up the Route
Finally, we need to register the `/login` route in the application router. 
Open `apps/issue-tracker/src/app/app.routes.ts` and add the route:

```typescript
// ...
  {
    path: 'login',
    loadComponent: () => import('@enterprise-workspace/shared/feature-auth').then(m => m.LoginComponent)
  },
// ...
```

### 5. Check Your Work!
1. Save all your files and navigate your browser to `http://localhost:4200/login`.
2. You should see your brand new Login page! 
3. Select a role and click **Login**. You will instantly be redirected to the Dashboard! 
4. Check your LocalStorage (F12 -> Application -> Local Storage). You will see your simulated JWT token safely stored there!
